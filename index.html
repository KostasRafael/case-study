Purpose

I built this project as an exercise for my CareerFoundry course to test and practice both, my back-end and front-end abilities.

Objective

To build an application completely from scratch, server side and client side. This will be part of my portfolio.

Server side

The back-end of this application consists of a restful API built with Node.js and a non-relational MongoDB database. The API gets it's data from the database, and exposes those data to users through it's various endpoints.

Client-side

The user interface was built using React alongside with the Redux state management library.

Retrospective

I enjoyed building everything from scratch, the databse, the restfull API and the user-interface, and connecting all these together. It was fascinating seeing the basic components of an application and how they connect with each other.

I struggled with debugging because I had to deal with the restfull API and the user interface. At one point, the UI displayed incorrect data, which could have stemmed from either the API response or how React handled it. I feared spending ages searching both. Fortunately, Postman helped by allowing me to send HTTP requests and isolate the API for testing. This revealed the issue was API-related, enabling me to fix it without any unnecessary front-end debugging..

My future plan is to connect my application to the IMDB API and pull movies from it, allowing my application to access a larger selection of movies.
