Case Study

My full stack project

myFlix

Overview

This is a full-stack web application encompassing both the front-end and back-end. The back-end is built using Node.js, leveraging an API that interacts with a MongoDB database to handle data storage and retrieval. The front-end is developed using both React.js enabling dynamic and responsive user interfaces. The API serves as the communication layer, facilitating seamless data exchange between the database and the client applications through structured HTTP requests.

The objective

To have a fully functional user interface that displays a list of movies. The user will be able to create an account, login, add movies to their list of favourite movies, update their profile information and delete their account

The challenge

To build all parts required for a fully functional application. Namely, the back-end infustructure that ensures data flow, and the front-end infastructure that ensure data consumption and usage through a seamless user interface

The Solution

Server side

The RESTful API was built with Node.js in combination with the following technologies:

  • Express.js
  • Passport
  • JWT
  • OAth2
  • CORS
  • Body-Parser

The database was build with non-relational database techniques using MongoDB and storing data in object-like structures. The data stored in the MongoDB are the movies data and the users data.

Client-side

The user interface was built using React alongside with:

  • React Redux
  • React-Bootstrap
  • Parcel
  • Brouser-Router

Using these technologies, I created different React components and then rendered those components by means of props. Also, I managed the state of the application with Redux, in a central store.

Retrospective

I enjoyed building everything from scratch, the databse, the restfull API and the user-interface, and connecting all these together. It was fascinating seeing the basic components of an application and how they connect with each other.

I struggled with debugging because I had to deal with the restfull API and the user interface. At one point, the UI displayed incorrect data, which could have stemmed from either the API response or how React handled it. I feared spending ages searching both. Fortunately, Postman helped by allowing me to send HTTP requests and isolate the API for testing. This revealed the issue was API-related, enabling me to fix it without any unnecessary front-end debugging..

My future plan is to connect my application to the IMDB API and pull movies from it, allowing my application to access a larger selection of movies.

Overall, I really enjoyed building this application. I’m super happy and proud to have created my own application entirely from scratch!
